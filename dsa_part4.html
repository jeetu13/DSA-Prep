Part 4 Overview
Trees are recursion made visible. Almost every tree problem is: "do something at this node, then recurse left and right." Master that and you've mastered trees.

🌟 The Universal Tree Template

// 90% of tree problems fit this shape:
// return solve(node):
//   if node == null → return base_value (null, 0, true, etc.)
//   left  = solve(node.left)

EasyDFS Inorder

1. Binary Tree Inorder Traversal (Recursive + Iterative)

▼
🧠 Logic Building
Inorder = Left → Root → Right. On a BST this visits nodes in ascending sorted order — the most important property in tree problems.
Recursive: trivial — go left, process, go right.
Iterative: Simulate the call stack manually. Push nodes while going left. When null, pop (= "returning from left subtree"), process, then go right.
Pattern unlock: "Sorted BST traversal / simulate recursion" → Stack-based inorder
Tree
4
  / \
 2  6
/ \
1  3
Inorder
[1, 2, 3, 4, 6]
// DRY RUN iterative on tree above:
// curr=4, stack=[]
// Push 4→2→1 (go left). curr=null.
// Pop 1 → add 1. curr=1.right=null.
// Pop 2 → add 2. curr=2.right=3.
// Push 3. curr=null.
// Pop 3 → add 3. curr=3.right=null.
// Pop 4 → add 4. curr=4.right=6.
// Push 6. curr=null.
// Pop 6 → add 6.
// Result: [1,2,3,4,6] ✓

// Recursive — 3 lines
public void inorder(TreeNode node, List<Integer> res) {
    if (node == null) return;
    inorder(node.left, res);
    res.add(node.val);   // process between children
    inorder(node.right, res);
}

// Iterative — O(1) extra space conceptually (stack replaces call stack)
public List<Integer> inorderIterative(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    Deque<TreeNode> st = new ArrayDeque<>();
    TreeNode curr = root;
    while (curr != null || !st.isEmpty()) {
        while (curr != null) { st.push(curr); curr = curr.left; } // go left
        curr = st.pop();           // backtrack
        res.add(curr.val);         // process
        curr = curr.right;           // go right
    }
    return res;
}
Time: O(N)Space: O(H) H=height
MediumBFS Level

2. Binary Tree Level Order Traversal (BFS)

▼
🧠 Logic Building
BFS visits nodes level by level. Use a Queue (FIFO). The key trick: snapshot queue.size() before the inner loop so you process exactly one level per outer iteration.
Variations built on this same template: zigzag level order (alternate reverse), right side view (take last of each level), level averages, min depth.
Pattern unlock: "Level-by-level processing" → Queue BFS + size snapshot
Tree
3 / 9,20 / _,_,15,7
Output
[[3],[9,20],[15,7]]
// DRY RUN: 3→[9,20]→[15,7]
// queue=[3]. Outer loop:
//   size=1: poll 3→level=[3], offer 9,20. res=[[3]]
//   size=2: poll 9→[9] (no children), poll 20→[9,20], offer 15,7. res=[[3],[9,20]]
//   size=2: poll 15,7→[15,7]. res=[[3],[9,20],[15,7]] ✓

public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int size = q.size(); // snapshot: process exactly this many (one level)
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode n = q.poll();
            level.add(n.val);
            if (n.left  != null) q.offer(n.left);
            if (n.right != null) q.offer(n.right);
        }
        res.add(level);
    }
    return res;
}
Time: O(N)Space: O(N)
MediumBFS Last of Level

3. Binary Tree Right Side View

▼
🧠 Logic Building
What's visible from the right? The last node at each level. BFS level-order is perfect — just take the last node of each level's inner loop.
DFS alternative: Traverse right subtree before left. Add current node if depth == res.size() (first visit at this depth from the right).
Pattern unlock: "Last/first node per level" → BFS level-order, pick boundary node
Tree
1→[2,3]→[_,5,_,4]
Right View
[1, 3, 4]
// DRY RUN: Level 0→[1], Level 1→[2,3], Level 2→[5,4]
// Last of each level: 1, 3, 4 ✓

public List<Integer> rightSideView(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int size = q.size();
        for (int i = 0; i < size; i++) {
            TreeNode n = q.poll();
            if (i == size - 1) res.add(n.val); // last node of this level
            if (n.left  != null) q.offer(n.left);
            if (n.right != null) q.offer(n.right);
        }
    }
    return res;
}
Time: O(N)Space: O(N)
EasyMirror DFS

4. Symmetric Tree — Mirror Check

▼
🧠 Logic Building
A tree is symmetric if it's a mirror of itself. Instead of checking a single node, compare two nodes simultaneously — the left subtree of the root vs the right subtree.
Mirror recursion: isMirror(L, R) = L.val==R.val AND isMirror(L.left, R.right) AND isMirror(L.right, R.left). Note the cross-comparison.
Base cases: Both null → true. One null → false.
Pattern unlock: "Tree symmetry" → Two-pointer recursion comparing opposite sides
Input
1→[2,2]→[3,4,4,3]
Output
true
// DRY RUN:
//       1
//      / \
//     2   2
//    / \ / \
//   3  4 4  3
// isMirror(2L,2R): vals match(2==2)
//   isMirror(3,3): ✓ both same
//   isMirror(4,4): ✓ both same → true ✓

public boolean isSymmetric(TreeNode root) {
    return mirror(root.left, root.right);
}
private boolean mirror(TreeNode L, TreeNode R) {
    if (L == null && R == null) return true;
    if (L == null || R == null) return false;
    return L.val == R.val
        && mirror(L.left,  R.right)  // outer pair
        && mirror(L.right, R.left);   // inner pair
}
Time: O(N)Space: O(H)

EasyPostorder Height

5. Maximum Depth of Binary Tree

▼
🧠 Logic Building
Definition: Depth = number of nodes on the longest root-to-leaf path.
Recursive formula: depth(node) = 1 + max(depth(left), depth(right)). Classic postorder — you need children's depths first before you can compute current node's depth.
Base case: null node has depth 0.
Pattern unlock: "Compute property bottom-up" → Postorder: return from leaves, combine at root
Tree
3→[9,20]→[_,_,15,7]
Output
3
// DRY RUN:
//       3          depth=3
//      / \
//     9   20       depth=2 for right subtree
//        / \
//       15  7      depth=1
//
// depth(15)=1, depth(7)=1
// depth(20)=1+max(1,1)=2
// depth(9)=1+max(0,0)=1
// depth(3)=1+max(1,2)=3 ✓

public int maxDepth(TreeNode root) {
    if (root == null) return 0;                            // base
    return 1 + Math.max(maxDepth(root.left), maxDepth(root.right)); // combine
}
Time: O(N)Space: O(H)
EasyReturn vs Update Global

6. Diameter of Binary Tree

▼
🧠 Logic Building
Diameter = longest path between any two nodes. This path may or may not pass through the root.
Key insight: For every node, the longest path THROUGH it = height(left) + height(right). The diameter is the maximum of this value across all nodes.
Two jobs at once: The recursive function RETURNS height (for the parent to use), but also UPDATES a global max diameter. This dual-purpose pattern appears in many hard tree problems.
Pattern unlock: "Path through current node = left-height + right-height" → Return height, update global max
Tree
1→[2,3]→[4,5,_,_]
Output
3 (path 4→2→1→3 or 5→2→1→3)
// DRY RUN: diameter=0 initially
//       1
//      / \
//     2   3
//    / \
//   4   5
//
// height(4)=1, height(5)=1
// at node 2: diameter = max(0, 1+1) = 2. return height=2
// height(3)=1
// at node 1: diameter = max(2, 2+1) = 3. return height=3
// return diameter=3 ✓

int diameter = 0; // global max, updated at each node

public int diameterOfBinaryTree(TreeNode root) {
    height(root);
    return diameter;
}
private int height(TreeNode node) {
    if (node == null) return 0;
    int L = height(node.left), R = height(node.right);
    diameter = Math.max(diameter, L + R); // path through current node
    return 1 + Math.max(L, R);            // height returned to parent
}
Time: O(N)Space: O(H)
EasyDFS Pass-Down

7. Path Sum — Root to Leaf

▼
🧠 Logic Building
Pass remaining sum DOWN the tree. At each node, subtract node.val from target. At a leaf, check if remaining == 0.
Leaf check: A leaf has no children. Don't return true at an internal node even if sum hits 0.
OR logic: Return true if LEFT path OR RIGHT path has the sum. Recursion handles this naturally.
Pattern unlock: "Root-to-leaf constraint" → DFS passing remaining value, check at leaf
Tree / Target
5→[4,8]→[11,_,13,4]→[7,2,_,_,_,1], target=22
Output
true (5→4→11→2)
// DRY RUN: target=22, path 5→4→11→2
// hasPath(5,22): rem=22-5=17, try left
//   hasPath(4,17): rem=17-4=13, try left
//     hasPath(11,13): rem=13-11=2, try left
//       hasPath(7,2): leaf! 2-7=-5≠0 → false
//     try right: hasPath(2,2): leaf! 2-2=0 → true ✓

public boolean hasPathSum(TreeNode root, int target) {
    if (root == null) return false;
    if (root.left == null && root.right == null)   // leaf node
        return root.val == target;
    return hasPathSum(root.left,  target - root.val)
        || hasPathSum(root.right, target - root.val);
}
Time: O(N)Space: O(H)
HardGlobal Max + Local Return

8. Binary Tree Maximum Path Sum

▼
🧠 Logic Building
Path can go through any nodes, not just root-to-leaf. It can start anywhere and end anywhere — but it must be a connected path (no branching in both directions simultaneously).
At each node, two things: (1) The "gain" this node contributes to its parent = node.val + max(leftGain, rightGain, 0). Can't use both branches when reporting UP. (2) The path THROUGH this node = node.val + leftGain + rightGain — update global max.
Why max(..., 0)? If a subtree gives negative gain, it's better to ignore it (don't include it in the path).
Pattern unlock: "Any-to-any path max sum" → Return single-branch gain up, track two-branch sum as global max
Tree
-10→[9,20]→[_,_,15,7]
Output
42 (15→20→7)
// DRY RUN:
//     -10
//     / \
//    9   20
//       / \
//      15   7
//
// gain(9)=9, gain(15)=15, gain(7)=7
// at node 20: path=20+15+7=42 → max=42. return 20+max(15,7)=35
// at node -10: path=-10+9+35=34 < 42. return max stays 42 ✓

int maxSum = Integer.MIN_VALUE;

public int maxPathSum(TreeNode root) {
    gain(root);
    return maxSum;
}
private int gain(TreeNode node) {
    if (node == null) return 0;
    int L = Math.max(gain(node.left),  0); // ignore negative subtrees
    int R = Math.max(gain(node.right), 0);
    maxSum = Math.max(maxSum, node.val + L + R); // path through this node
    return node.val + Math.max(L, R);            // best single branch to parent
}
Time: O(N)Space: O(H)
MediumLCA DFS

9. Lowest Common Ancestor — Binary Tree

▼
🧠 Logic Building
LCA = deepest node that has both p and q as descendants.
Key insight: When recursing, if a node IS p or q → return it immediately (it's either the LCA or a descendant to report). Don't recurse deeper once found.
Combining results: If both left and right return non-null, current node = LCA (p is in one subtree, q in the other). If only one side returns non-null, bubble that up.
Pattern unlock: "First node where both targets split to different subtrees" → Return found nodes up, LCA = first split point
Tree, p=5, q=1
3→[5,1]→[6,2,0,8]
Output
3
// DRY RUN: find LCA(5,1) in tree rooted at 3
// lca(3): recurse left(5), right(1)
//   lca(5): node==p → return 5 (don't go deeper)
//   lca(1): node==q → return 1
// At node 3: left=5 (non-null), right=1 (non-null) → both found → LCA=3 ✓
//
// LCA(5,4) where 4 is under 5:
// lca(5): node==p → return 5 immediately (4 is under 5 but we return here)
// At 3: left=5, right=null → return 5 (LCA is 5 itself) ✓

public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;
    TreeNode left  = lowestCommonAncestor(root.left,  p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);
    if (left != null && right != null) return root;  // split here → LCA
    return left != null ? left : right;              // bubble found node up
}
Time: O(N)Space: O(H)
MediumPreorder In-Place

10. Flatten Binary Tree to Linked List

▼
🧠 Logic Building
Preorder = root → left → right. Flattened list follows preorder. So after flattening: each node's right = its preorder successor, left = null.
Trick (Morris-like): For each node, if it has a left child: find the rightmost node of the left subtree, attach current right there, then move left subtree to right, clear left.
This is O(1) space — no recursion stack needed.
Pattern unlock: "Rewire tree in preorder, in-place" → Find left-subtree tail, splice
Tree
1→[2,5]→[3,4,_,6]
Output
1→2→3→4→5→6 (right pointers only)
// DRY RUN:
//     1           After step 1 (node=1):
//    / \          rightmost of left(2)=4
//   2   5         4.right = 5 (old right)
//  / \ / \    →   1.right = 2, 1.left=null
// 3  4 _ 6
//                 1→2→3→4→5→6 (continuing...) ✓

public void flatten(TreeNode root) {
    TreeNode curr = root;
    while (curr != null) {
        if (curr.left != null) {
            TreeNode tail = curr.left;
            while (tail.right != null) tail = tail.right; // find tail of left
            tail.right = curr.right; // connect old right to tail
            curr.right = curr.left;  // move left subtree to right
            curr.left  = null;
        }
        curr = curr.right;
    }
}
Time: O(N)Space: O(1)
MediumDivide by Root Index

11. Construct Tree from Preorder + Inorder

▼
🧠 Logic Building
Preorder[0] is always the root. Find that root in inorder — everything LEFT of it in inorder = left subtree, everything RIGHT = right subtree.
Recursively: Build left subtree using preorder[1..leftSize] and inorder[0..rootIdx-1]. Build right subtree from the rest.
Optimization: Use a HashMap for inorder index lookups → O(1) per call instead of O(N) scan.
Pattern unlock: "Preorder gives root, inorder gives subtree boundaries" → Divide by root's inorder index
Input
preorder=[3,9,20,15,7], inorder=[9,3,15,20,7]
Output
Tree: 3→[9,20]→[_,_,15,7]
// DRY RUN: preorder=[3,9,20,15,7], inorder=[9,3,15,20,7]
// root=preorder[0]=3. In inorder: 3 is at idx 1.
//   left subtree: inorder[0..0]=[9], preorder[1..1]=[9]
//     root=9, no children
//   right subtree: inorder[2..4]=[15,20,7], preorder[2..4]=[20,15,7]
//     root=20. In [15,20,7]: 20 at idx 1.
//       left: [15], right: [7]
// Result tree: 3→[9,20]→[_,_,15,7] ✓

public TreeNode buildTree(int[] pre, int[] ino) {
    Map<Integer,Integer> map = new HashMap<>();
    for (int i=0; i<ino.length; i++) map.put(ino[i], i); // O(1) index lookup
    return build(pre, 0, pre.length-1, ino, 0, ino.length-1, map);
}
private TreeNode build(int[] pre, int pl, int pr, int[] ino, int il, int ir, Map<Integer,Integer> m) {
    if (pl > pr) return null;
    TreeNode root = new TreeNode(pre[pl]);
    int ri = m.get(pre[pl]);     // root's position in inorder
    int leftSize = ri - il;
    root.left  = build(pre, pl+1, pl+leftSize, ino, il, ri-1, m);
    root.right = build(pre, pl+leftSize+1, pr, ino, ri+1, ir, m);
    return root;
}
Time: O(N)Space: O(N)

MediumValid Range DFS

12. Validate Binary Search Tree

▼
🧠 Logic Building
Common mistake: Only checking node.val > node.left.val is WRONG. A right descendant of a left child could violate the BST property.
Correct approach: Each node must be within a valid range [min, max]. Root: (-∞, +∞). Left child: (min, root.val). Right child: (root.val, max).
Pass the valid range DOWN: At each node, check current value is within range, then recurse with tightened bounds.
Pattern unlock: "BST validation needs global context, not just local comparison" → Pass min/max bounds down recursively
Invalid BST
5→[1,4]→[_,_,3,6] (4 is right of 5 but 3<5)
Output
false
// DRY RUN: 5→[1,4]
// isValid(5, -∞, +∞): 5 in range ✓
//   isValid(1, -∞, 5): 1 in range ✓
//   isValid(4, 5, +∞): 4 < min=5 → FALSE ✓
//
// DRY RUN valid: 2→[1,3]
// isValid(2, -∞, +∞) → isValid(1,-∞,2) ✓ → isValid(3,2,+∞) ✓ → true ✓

public boolean isValidBST(TreeNode root) {
    return valid(root, Long.MIN_VALUE, Long.MAX_VALUE);
}
private boolean valid(TreeNode node, long min, long max) {
    if (node == null) return true;
    if (node.val <= min || node.val >= max) return false; // out of valid range
    return valid(node.left,  min,      node.val)   // left: upper bound tightens
        && valid(node.right, node.val, max);         // right: lower bound tightens
}
Time: O(N)Space: O(H)
EasyBST Navigate

13. Search in a BST

▼
🧠 Logic Building
BST gives you a direction at every node: val < root → go left. val > root → go right. val == root → found.
This eliminates half the tree at each step → O(log N) average.
Pattern unlock: "Binary decision at each node" → O(log N) navigation using BST ordering
BST, val=2
4→[2,7]→[1,3,_,_]
Output
Subtree rooted at 2
// DRY RUN: search(4, val=2)
//   2 < 4 → go left to node 2
//   2 == 2 → found! return node(2) ✓

public TreeNode searchBST(TreeNode root, int val) {
    while (root != null) {
        if      (val == root.val) return root;
        else if (val <  root.val) root = root.left;
        else                      root = root.right;
    }
    return null; // not found
}
Time: O(log N) avgSpace: O(1)
MediumBST Insert

14. Insert into a BST

▼
🧠 Logic Building
Navigate like search, but insert at the first null you reach. The new value always goes to a leaf position (no restructuring needed for insertion).
Return the modified tree root — recursive approach returns the node at each step to reconnect the tree.
Pattern unlock: "Search for null position, attach new node there" → BST recursive insert
BST, insert=5
4→[2,7]→[1,3,_,_]
Output
4→[2,7]→[1,3,5,_]
// DRY RUN: insert 5 into BST rooted at 4
// 5 > 4 → go right to 7
// 5 < 7 → go left. 7.left = null
// insert here: 7.left = new TreeNode(5) ✓

public TreeNode insertIntoBST(TreeNode root, int val) {
    if (root == null) return new TreeNode(val); // found insertion point
    if (val < root.val) root.left  = insertIntoBST(root.left,  val);
    else               root.right = insertIntoBST(root.right, val);
    return root; // return current node to rebuild parent links
}
Time: O(log N) avgSpace: O(H)
MediumBST Delete 3 Cases

15. Delete Node from BST

▼
🧠 Logic Building
3 cases when deleting a node:
Case 1 — Leaf (no children): Just remove it. Return null to parent.
Case 2 — One child: Return the existing child to the parent. The child "replaces" the deleted node.
Case 3 — Two children: Find the inorder successor (smallest in right subtree). Copy its value to current node. Delete the inorder successor from the right subtree.
Pattern unlock: "Replace with inorder successor when two children" → Find min of right subtree, swap values, delete successor
BST, delete 3
5→[3,6]→[2,4,_,7]
Output
5→[4,6]→[2,_,_,7]
// DRY RUN: delete 3 (has 2 children: 2 and 4)
// Find inorder successor = min of right subtree = 4
// Replace node(3).val with 4
// Delete 4 from right subtree of node (now node.val=4)
// Result: 5→[4,6]→[2,_,_,7] ✓

public TreeNode deleteNode(TreeNode root, int key) {
    if (root == null) return null;
    if (key < root.val) root.left  = deleteNode(root.left,  key);
    else if (key > root.val) root.right = deleteNode(root.right, key);
    else {                                         // found node to delete
        if (root.left  == null) return root.right; // case 1 or 2
        if (root.right == null) return root.left;  // case 2
        TreeNode succ = root.right;                // case 3: find inorder successor
        while (succ.left != null) succ = succ.left;
        root.val   = succ.val;                     // copy successor's value
        root.right = deleteNode(root.right, succ.val); // delete successor
    }
    return root;
}
Time: O(log N) avgSpace: O(H)
MediumInorder Counter

16. Kth Smallest Element in BST

▼
🧠 Logic Building
BST inorder = sorted ascending. Kth smallest = kth element in inorder traversal.
Don't collect all then index — stop early as soon as you've visited k nodes. Use a counter and return immediately.
Pattern unlock: "Kth in BST = kth inorder" → Inorder with early-stop counter
BST, k=3
3→[1,4]→[_,2,_,_]
Output
3 (inorder: 1,2,3,4)
// DRY RUN: k=3, BST: 3→[1,4]→[_,2]
// Inorder: visit 1(k=2→1), visit 2(k=1→0 wait no, count up)
// count=0: visit 1 → count=1, visit 2 → count=2, visit 3 → count=3==k → return 3 ✓

int count = 0, result = 0;
public int kthSmallest(TreeNode root, int k) {
    inorder(root, k);
    return result;
}
private void inorder(TreeNode node, int k) {
    if (node == null) return;
    inorder(node.left, k);
    if (++count == k) { result = node.val; return; } // found kth
    inorder(node.right, k);
}
Time: O(H+K)Space: O(H)
MediumBST Navigate to Split

17. LCA of a BST

▼
🧠 Logic Building
BST LCA is simpler than general tree LCA because you can navigate by value.
Key insight: If both p and q are less than root → LCA is in left subtree. If both greater → right subtree. If they split (one each side, or one equals root) → root IS the LCA.
No need for general DFS — navigate directly to the split point.
Pattern unlock: "BST: split point = LCA" → Navigate by comparing both values to current node
BST, p=2, q=8
6→[2,8]→[0,4,7,9]
Output
6
// DRY RUN: p=2, q=8, root=6
// 2 < 6 AND 8 > 6 → they're on different sides → LCA = 6 ✓
//
// DRY RUN 2: p=2, q=4, root=6
// Both < 6 → go left to node 2
// p=2 == root=2 → LCA = 2 ✓

public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    while (root != null) {
        if (p.val < root.val && q.val < root.val)
            root = root.left;            // both in left subtree
        else if (p.val > root.val && q.val > root.val)
            root = root.right;           // both in right subtree
        else
            return root;               // split point = LCA
    }
    return null;
}
Time: O(H)Space: O(1)

18. Serialize and Deserialize Binary Tree

▼
🧠 Logic Building
Serialize = tree → string. Deserialize = string → tree. Must handle null children to reconstruct the exact structure.
Use preorder DFS for serialization. Use a special marker (e.g. "#") for null nodes. This uniquely identifies the tree structure — preorder alone (without inorder) IS enough when nulls are explicitly marked.
Deserialize: Use a queue or index pointer on the tokens. Recursively reconstruct: take next token, if "#" return null, else create node and recurse for left then right.
Why preorder? Root comes first — you can create the root node immediately, then recurse for children. Postorder or inorder would require extra lookahead.
Pattern unlock: "Encode structure with null markers" → Preorder DFS with "#" for null, queue for deserialization
Tree
1→[2,3]→[_,_,4,5]
Serialized
"1,2,#,#,3,4,#,#,5,#,#"
// SERIALIZE DRY RUN: tree 1→[2,3]→[_,_,4,5]
// preorder: visit 1 → "1,"
//   go left  → 2 → "1,2,"
//     go left  → null → "1,2,#,"
//     go right → null → "1,2,#,#,"
//   go right → 3 → "1,2,#,#,3,"
//     go left  → 4 → "...3,4,"
//       null,null → "...4,#,#,"
//     go right → 5 → "...5,#,#"
// Final: "1,2,#,#,3,4,#,#,5,#,#" ✓
//
// DESERIALIZE: queue = [1,2,#,#,3,4,#,#,5,#,#]
// poll 1 → node(1). recurse left:
//   poll 2 → node(2). recurse left: poll # → null. recurse right: poll # → null.
//   return node(2)
// recurse right: poll 3 → node(3). recurse left:
//   poll 4 → node(4). #,# → null,null. return node(4)
// recurse right: poll 5 → node(5). #,# → return node(5)
// Tree reconstructed ✓

public String serialize(TreeNode root) {
    StringBuilder sb = new StringBuilder();
    serDFS(root, sb);
    return sb.toString();
}
private void serDFS(TreeNode n, StringBuilder sb) {
    if (n == null) { sb.append("#,"); return; }
    sb.append(n.val).append(',');
    serDFS(n.left, sb); serDFS(n.right, sb);
}

public TreeNode deserialize(String data) {
    Queue<String> q = new LinkedList<>(Arrays.asList(data.split(",")));
    return desDFS(q);
}
private TreeNode desDFS(Queue<String> q) {
    String val = q.poll();
    if ("#".equals(val)) return null;
    TreeNode node = new TreeNode(Integer.parseInt(val));
    node.left  = desDFS(q);
    node.right = desDFS(q);
    return node;
}
Time: O(N)Space: O(N)
