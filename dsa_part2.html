1. Reverse Linked List

▼
🧠 Logic Building
Visualize: 1→2→3→null. We want null←1←2←3. Each node's next pointer needs to flip.
Problem: If you flip curr.next, you lose the rest of the list. Solution: save next BEFORE flipping.
3-pointer dance: prev=null, curr=head. At each step: save curr.next, flip curr.next=prev, advance prev=curr, curr=saved.
Termination: When curr=null, prev points to the new head.
Pattern unlock: "Reverse pointer direction" → 3-pointer iterative (prev, curr, next)
Input
1→2→3→4→5→null
Output
5→4→3→2→1→null
// DRY RUN: 1->2->3->null
// Initial: prev=null, curr=1
// Step 1: next=2, curr(1).next=null, prev=1, curr=2   → null<-1  2->3
// Step 2: next=3, curr(2).next=1,   prev=2, curr=3   → null<-1<-2  3->null
// Step 3: next=null, curr(3).next=2, prev=3, curr=null → null<-1<-2<-3
// curr==null → return prev=3 ✓

public ListNode reverseList(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next; // 1. save next before breaking link
        curr.next = prev;          // 2. flip the pointer
        prev = curr;               // 3. advance prev
        curr = next;               // 4. advance curr
    }
    return prev; // prev is new head when curr==null
}
Time: O(N)Space: O(1)
EasyFloyd's Fast/Slow

2. Linked List Cycle Detection

▼
🧠 Logic Building
Problem: Detect if a linked list has a cycle. HashSet approach: O(N) space. Can we do O(1)?
Analogy: Imagine two runners on a circular track. The faster one will always lap the slower one — they'll meet. No cycle = fast runner reaches the end.
Algorithm: slow moves 1 step, fast moves 2 steps. If they ever meet → cycle. If fast hits null → no cycle.
Cycle start: After meeting, move one pointer back to head. Advance both 1 step at a time. They meet at the cycle start node.
Pattern unlock: "Two things moving at different speeds" → Fast/slow pointers (Floyd's)
Input
3→2→0→-4→(back to 2)
Output
true, cycle at node 2
// DRY RUN: 3->2->0->-4->2 (cycle at index 1)
// slow=3, fast=3
// Step 1: slow=2, fast=0
// Step 2: slow=0, fast=2  (fast: -4->2)
// Step 3: slow=-4, fast=-4 → MEET! cycle detected ✓
//
// Find cycle start: reset slow=head(3), keep fast=-4
// Step 1: slow=2, fast=2 → MEET at node 2 = cycle start ✓

public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true; // pointers met → cycle
    }
    return false;
}

// BONUS: Find cycle start node
public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next; fast = fast.next.next;
        if (slow == fast) {
            slow = head; // reset one to head
            while (slow != fast) { slow = slow.next; fast = fast.next; }
            return slow; // cycle start
        }
    }
    return null;
}
Time: O(N)Space: O(1)

3. Find Middle of Linked List

▼
🧠 Logic Building
Naive: Count N nodes, then walk N/2 steps = 2 passes.
Insight: When fast (2x speed) reaches the end, slow (1x speed) is exactly halfway. One pass!
Odd vs even: fast!=null gives second middle for even lists. fast.next!=null gives first middle.
Pattern unlock: "Find N/2 position in one pass" → Fast/slow where fast=2×slow
Input
1→2→3→4→5
Output
Node 3 (middle)
// DRY RUN: 1->2->3->4->5
// slow=1,fast=1
// Step 1: slow=2, fast=3
// Step 2: slow=3, fast=5
// Step 3: fast.next=null → stop. return slow=3 ✓
//
// Even: 1->2->3->4
// Step 1: slow=2, fast=3
// Step 2: slow=3, fast=null(fast was 3, fast.next=4, fast.next.next=null)
// Actually: fast=4 after step 1, fast.next=null → stop at slow=3 (second middle) ✓

public ListNode middleNode(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;       // moves 1 step
        fast = fast.next.next;  // moves 2 steps
    }
    return slow; // at N/2 when fast reaches end
}
Time: O(N)Space: O(1)
EasyDummy Node + Merge

4. Merge Two Sorted Linked Lists

▼
🧠 Logic Building
Think merge sort's merge step: Compare heads, take smaller, advance that pointer.
Dummy node trick: Create a fake head node. This eliminates special-casing the first element — you always append to dummy.next.
Leftover: When one list exhausts, append the other directly (already sorted).
Pattern unlock: "Build result list incrementally" → Dummy head + pointer weaving
Input
l1=1→2→4, l2=1→3→4
Output
1→1→2→3→4→4
// DRY RUN: l1=1->2->4, l2=1->3->4
// dummy=0, curr=dummy
// l1(1) == l2(1): take l1(1), curr->1, l1=2
// l1(2) >  l2(1): take l2(1), curr->1->1, l2=3
// l1(2) <  l2(3): take l1(2), curr->1->1->2, l1=4
// l1(4) >  l2(3): take l2(3), curr->...->3, l2=4
// l1(4) == l2(4): take l1(4), l1=null
// l1=null: append l2(4) directly
// return dummy.next = 1->1->2->3->4->4 ✓

public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0), curr = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) { curr.next = l1; l1 = l1.next; }
        else                   { curr.next = l2; l2 = l2.next; }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2; // attach remaining
    return dummy.next;
}
Time: O(M+N)Space: O(1)
MediumTwo Pointers Gap

5. Remove Nth Node from End

▼
🧠 Logic Building
Naive: Count total N, then walk to (N - nth) position. Two passes.
One-pass insight: Maintain two pointers exactly n steps apart. When fast hits the end, slow is at the node BEFORE the one to remove.
Gap trick: Advance fast n steps first. Then move both together until fast.next=null.
Dummy node: Handles edge case of removing the head (n == list length).
Pattern unlock: "Nth from end in one pass" → Two pointers with n-step gap
Input
1→2→3→4→5, n=2
Output
1→2→3→5
// DRY RUN: 1->2->3->4->5, n=2
// dummy->1->2->3->4->5, slow=fast=dummy
// Advance fast n=2 steps: fast=1, fast=2
// Move both until fast.next=null:
//   fast(2).next=3≠null: slow=1, fast=3
//   fast(3).next=4≠null: slow=2, fast=4
//   fast(4).next=5≠null: slow=3, fast=5
//   fast(5).next=null → stop
// slow=3, slow.next=4 (to remove). slow.next=4.next=5
// List: 1->2->3->5 ✓

public ListNode removeNthFromEnd(ListNode head, int n) {
    ListNode dummy = new ListNode(0, head);
    ListNode slow = dummy, fast = dummy;
    for (int i = 0; i < n; i++) fast = fast.next; // advance fast n steps
    while (fast.next != null) { slow = slow.next; fast = fast.next; }
    slow.next = slow.next.next; // skip the nth node
    return dummy.next;
}
Time: O(N)Space: O(1)

6. Palindrome Linked List

▼
🧠 Logic Building
If it were an array: Two pointers from ends, compare. Easy. But LL has no backward traversal.
Key idea: Find middle (fast/slow), reverse the second half in-place, compare both halves from the front.
Steps: (1) Find mid using fast/slow. (2) Reverse from mid.next. (3) Compare head vs reversed half. (4) Optionally restore.
Pattern unlock: "Palindrome check on LL" → Find mid + reverse second half + compare
Input
1→2→2→1
Output
true
// DRY RUN: 1->2->2->1
// Step 1 find mid: slow=2(idx1), fast=1(idx3)
// Step 2 reverse second half from slow.next=2->1:
//   reversed: 1->2->null
// Step 3 compare: head=1 vs rev=1 ✓, head=2 vs rev=2 ✓ → true ✓

public boolean isPalindrome(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) { slow=slow.next; fast=fast.next.next; }
    ListNode rev = reverse(slow); // reverse second half
    ListNode p1 = head, p2 = rev;
    while (p2 != null) {
        if (p1.val != p2.val) return false;
        p1=p1.next; p2=p2.next;
    }
    return true;
}
private ListNode reverse(ListNode head) {
    ListNode prev=null, curr=head;
    while(curr!=null){ListNode nx=curr.next;curr.next=prev;prev=curr;curr=nx;}
    return prev;
}
Time: O(N)Space: O(1)

7. Valid Parentheses

▼
🧠 Logic Building
Observation: Every closing bracket must match the MOST RECENTLY seen unmatched opening bracket. That's exactly what a stack gives you — LIFO access to the most recent item.
Algorithm: Push opening brackets. On closing bracket, check if top of stack matches. If not → invalid.
End check: Stack must be empty at the end (no unmatched opens).
Pattern unlock: "Match recent open with current close" → Stack push/pop
Input
s = "()[]{}"
Output
true
Input 2
s = "([)]"
Output 2
false
// DRY RUN: "([)]"
//   '(': push → stack=['(']
//   '[': push → stack=['(','[']
//   ')': map[')'] = '('. pop → '[' ≠ '(' → return false ✓
//
// DRY RUN: "()[]{}"
//   '(': push=['(']
//   ')': pop='(', map[')']=('(') → match ✓, stack=[]
//   '[': push=['[']
//   ']': pop='[', match ✓, stack=[]
//   '{': push=['{']
//   '}': pop='{', match ✓, stack=[]
//   stack empty → return true ✓

public boolean isValid(String s) {
    Deque<Character> st = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if      (c=='('||c=='['||c=='{') st.push(c);
        else if (c==')' && (st.isEmpty()||st.pop()!='(') return false;
        else if (c==']' && (st.isEmpty()||st.pop()!='[') return false;
        else if (c=='}' && (st.isEmpty()||st.pop()!='{') return false;
    }
    return st.isEmpty(); // unmatched opens = invalid
}
Time: O(N)Space: O(N)
MediumAuxiliary Stack

8. Min Stack — O(1) getMin()

▼
🧠 Logic Building
Challenge: Standard stack gives O(1) push/pop/top but O(N) for minimum.
Insight: Maintain a parallel "min stack" that records the current minimum AT EACH LEVEL of the main stack. When you push x, also push min(x, minStack.top).
Why it works: When you pop from main stack, also pop from minStack. The min at any depth is always at the top of minStack.
Pattern unlock: "Track aggregate (min/max) through push/pop" → Parallel auxiliary stack
Operations
push(-2), push(0), push(-3), getMin(), pop(), top(), getMin()
Output
-3, 0, -2
// DRY RUN:
// push(-2): stack=[-2], minSt=[-2]
// push(0):  stack=[-2,0], minSt=[-2,-2]   ← min stays -2
// push(-3): stack=[-2,0,-3], minSt=[-2,-2,-3]  ← new min
// getMin(): minSt.top = -3 ✓
// pop():    stack=[-2,0], minSt=[-2,-2]
// top():    stack.top = 0 ✓
// getMin(): minSt.top = -2 ✓

class MinStack {
    Deque<Integer> stack = new ArrayDeque<>();
    Deque<Integer> minSt = new ArrayDeque<>(); // tracks min at each level
    public void push(int val) {
        stack.push(val);
        minSt.push(minSt.isEmpty() ? val : Math.min(val, minSt.peek()));
    }
    public void pop()        { stack.pop(); minSt.pop(); }
    public int  top()        { return stack.peek(); }
    public int  getMin()    { return minSt.peek(); }
}
Time: O(1) all opsSpace: O(N)

9. Evaluate Reverse Polish Notation

▼
🧠 Logic Building
RPN: Operands come before operator. "2 3 +" means 2+3=5. No parentheses needed.
Stack model: Push numbers. When you see an operator, pop two numbers, apply operator, push result. At the end, stack has one item: the answer.
Order matters: First pop = right operand. Second pop = left operand. (For subtraction/division this matters!)
Pattern unlock: "Evaluate expression with deferred operations" → Stack-based evaluation
Input
["2","1","+","3","*"]
Output
9  // (2+1)*3
// DRY RUN: ["2","1","+","3","*"]
//   "2": stack=[2]
//   "1": stack=[2,1]
//   "+": pop 1,2 → 2+1=3 → stack=[3]
//   "3": stack=[3,3]
//   "*": pop 3,3 → 3*3=9 → stack=[9]
//   return 9 ✓

public int evalRPN(String[] tokens) {
    Deque<Integer> st = new ArrayDeque<>();
    for (String t : tokens) {
        if ("+*-/".contains(t)) {
            int b=st.pop(), a=st.pop(); // b=right, a=left operand
            st.push(t.equals("+")?a+b:t.equals("-")?a-b:t.equals("*")?a*b:a/b);
        } else st.push(Integer.parseInt(t));
    }
    return st.pop();
}
Time: O(N)Space: O(N)

10. Daily Temperatures — Next Warmer Day

▼
🧠 Logic Building
Brute force: For each day, scan forward until warmer → O(N²).
Key insight: We want the NEXT GREATER element for each index. When we find a warmer day at j for unresolved day i, we can answer i immediately.
Stack stores indices of days waiting for their answer. It's monotonically decreasing in temperature (we never push if current is colder than top).
Process: For each day j, while stack top's temp < today's temp: pop, answer that day with (j - popped_index). Push j.
Pattern unlock: "Next greater element" → Monotonic decreasing stack of indices
Input
[73,74,75,71,69,72,76,73]
Output
[1,1,4,2,1,1,0,0]
// DRY RUN: [73,74,75,71,69,72,76,73]
//   i=0(73): stack=[] → push 0. stack=[0]
//   i=1(74): 74>temps[0]=73 → pop 0, res[0]=1-0=1. push 1. stack=[1]
//   i=2(75): 75>temps[1]=74 → pop 1, res[1]=2-1=1. push 2. stack=[2]
//   i=3(71): 71<75 → push 3. stack=[2,3]
//   i=4(69): 69<71 → push 4. stack=[2,3,4]
//   i=5(72): 72>temps[4]=69 → pop 4, res[4]=5-4=1
//             72>temps[3]=71 → pop 3, res[3]=5-3=2
//             72temps[5]=72 → pop 5, res[5]=6-5=1
//             76>temps[2]=75 → pop 2, res[2]=6-2=4
//             stack empty → push 6. stack=[6]
//   i=7(73): 73<76 → push 7. stack=[6,7]
//   Remaining in stack: res[6]=res[7]=0 (no warmer day)
//   return [1,1,4,2,1,1,0,0] ✓

public int[] dailyTemperatures(int[] temps) {
    int[] res = new int[temps.length];
    Deque<Integer> st = new ArrayDeque<>(); // stores indices
    for (int i = 0; i < temps.length; i++) {
        while (!st.isEmpty() && temps[i] > temps[st.peek()]) {
            int idx = st.pop();
            res[idx] = i - idx; // days waited = current - waiting index
        }
        st.push(i);
    }
    return res; // remaining in stack → res stays 0 (no warmer day)
}
Time: O(N)Space: O(N)

11. Implement Queue using Two Stacks

▼
🧠 Logic Building
Problem: Stack = LIFO. Queue = FIFO. They're opposites. Two opposites cancel each other out!
Two-stack trick: stack1=inbox (push here). stack2=outbox (pop from here). When outbox is empty, pour all of inbox into outbox — this reverses the order = FIFO!
Amortized O(1): Each element moves from inbox to outbox exactly once. Total work = 2N over N operations → O(1) amortized per operation.
Pattern unlock: "FIFO from LIFO" → Double-reverse with two stacks
// DRY RUN: push(1), push(2), pop(), push(3), pop()
// push(1): in=[1], out=[]
// push(2): in=[1,2], out=[]
// pop():   out empty → pour in→out: out=[2,1], in=[]
//          pop out → return 1 ✓ out=[2]
// push(3): in=[3], out=[2]
// pop():   out not empty → pop out → return 2 ✓ out=[]

class MyQueue {
    Deque<Integer> in=new ArrayDeque<>(), out=new ArrayDeque<>();
    public void push(int x) { in.push(x); }
    public int  pop()       { move(); return out.pop(); }
    public int  peek()      { move(); return out.peek(); }
    public boolean empty() { return in.isEmpty() && out.isEmpty(); }
    private void move() {
        if (out.isEmpty())                  // only pour when outbox empty
            while (!in.isEmpty()) out.push(in.pop());
    }
}
Time: O(1) amortizedSpace: O(N)

12. Binary Tree Level Order Traversal

▼
🧠 Logic Building
BFS template: Queue starts with root. Each iteration: process ALL nodes at current level (snapshot the queue size first!), add their children.
Level snapshot trick: int size = queue.size() before inner loop. Process exactly `size` nodes — that's one full level.
DFS vs BFS: DFS (recursion/stack) goes deep first. BFS (queue) goes wide first — naturally level-by-level.
Pattern unlock: "Process level by level" → BFS with level-size snapshot
Input
Tree: 3 / 9,20 / _,_,15,7
Output
[[3],[9,20],[15,7]]
// DRY RUN: tree with root=3, children=9,20; 20's children=15,7
// queue=[3]
// Level 1: size=1 → process 3 → level=[3], add 9,20. queue=[9,20]
// Level 2: size=2 → process 9 → level=[9], no children
//                 → process 20 → level=[9,20], add 15,7. queue=[15,7]
// Level 3: size=2 → process 15,7 → level=[15,7]. queue=[]
// return [[3],[9,20],[15,7]] ✓

public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int size = q.size(); // snapshot current level size
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = q.poll();
            level.add(node.val);
            if (node.left  != null) q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
        res.add(level);
    }
    return res;
}
Time: O(N)Space: O(N)

13. Rotting Oranges — Minimum Minutes

▼
🧠 Logic Building
Key insight: Multiple rotten oranges spread simultaneously. This screams Multi-Source BFS — start BFS from ALL rotten oranges at once.
Each BFS level = 1 minute. Count levels until no more fresh oranges can be reached.
End check: After BFS, if any fresh orange remains → return -1 (unreachable, surrounded by empty).
Pattern unlock: "Simultaneous spread from multiple sources" → Multi-source BFS
Input
[[2,1,1],[1,1,0],[0,1,1]]
Output
4
// 2=rotten, 1=fresh, 0=empty
// DRY RUN: [[2,1,1],[1,1,0],[0,1,1]]
// Init: queue=[(0,0,rotten)], fresh=6
// Min 1: rot (0,1),(1,0). fresh=4
// Min 2: rot (0,2),(1,1). fresh=2
// Min 3: no new reachable from (0,2)&(1,1)...
//         (1,1) rots (2,1). fresh=1
// Min 4: (2,1) rots (2,2). fresh=0
// return 4 ✓

public int orangesRotting(int[][] grid) {
    int rows=grid.length, cols=grid[0].length, fresh=0, minutes=0;
    Queue<int[]> q = new LinkedList<>();
    for(int r=0;r<rows;r++) for(int c=0;c<cols;c++){
        if(grid[r][c]==2) q.offer(new int[]{r,c}); // all rotten → queue
        else if(grid[r][c]==1) fresh++;
    }
    int[][] dirs={{0,1},{0,-1},{1,0},{-1,0}};
    while (!q.isEmpty() && fresh > 0) {
        minutes++;
        int size=q.size();
        for(int i=0;i<size;i++){
            int[] cell=q.poll();
            for(int[] d:dirs){
                int nr=cell[0]+d[0], nc=cell[1]+d[1];
                if(nr>=0&&nr<rows&&nc>=0&&nc<cols&&grid[nr][nc]==1){
                    grid[nr][nc]=2; fresh--; q.offer(new int[]{nr,nc});
                }
            }
        }
    }
    return fresh==0 ? minutes : -1;
}
Time: O(R×C)Space: O(R×C)

14. Next Greater Element I

▼
🧠 Logic Building
Brute force: For each element, scan right until greater found → O(N²).
Monotonic stack insight: Process right to left. Maintain a stack of candidates. For each element, pop smaller candidates (they'll never be "next greater" for anything to the left). Top = next greater.
Alternatively left to right: Push indices. When current > stack top → current is the NGE for top. Pop and record.
Pattern unlock: "NGE = first element to the right that's larger" → Monotonic decreasing stack
Input
nums1=[4,1,2], nums2=[1,3,4,2]
Output
[-1,3,-1]
// DRY RUN: nums2=[1,3,4,2], build NGE map
// Process left to right with monotonic decreasing stack:
//   i=0(1): stack=[] → push 1. stack=[1]
//   i=1(3): 3>stack top(1) → pop 1, nge[1]=3. stack=[] → push 3. stack=[3]
//   i=2(4): 4>stack top(3) → pop 3, nge[3]=4. stack=[] → push 4. stack=[4]
//   i=3(2): 2<4 → push 2. stack=[4,2]
//   End: remaining [4,2] have no NGE → nge[4]=-1, nge[2]=-1
// nge map: {1:3, 3:4, 4:-1, 2:-1}
// For nums1=[4,1,2]: lookup nge[4]=-1, nge[1]=3, nge[2]=-1
// return [-1,3,-1] ✓

public int[] nextGreaterElement(int[] nums1, int[] nums2) {
    Map<Integer,Integer> nge = new HashMap<>();
    Deque<Integer> st = new ArrayDeque<>();
    for (int n : nums2) {
        while (!st.isEmpty() && n > st.peek()) nge.put(st.pop(), n);
        st.push(n);
    }
    while (!st.isEmpty()) nge.put(st.pop(), -1); // no NGE found
    int[] res = new int[nums1.length];
    for (int i=0; i<nums1.length; i++) res[i] = nge.get(nums1[i]);
    return res;
}
Time: O(N+M)Space: O(N)

15. Largest Rectangle in Histogram

🧠 Logic Building
Key insight: For each bar, the largest rectangle using that bar's height extends leftward to the first shorter bar on its left and rightward to the first shorter bar on its right.
Monotonic increasing stack: When we pop a bar (because current is shorter), that bar's "right boundary" = current index. Its "left boundary" = new stack top + 1. Width = right - left - 1.
Sentinel trick: Append height 0 at the end to flush remaining stack items.
Pattern unlock: "Area bounded by first shorter on each side" → Monotonic increasing stack
Input
heights=[2,1,5,6,2,3]
Output
10  // bars 5,6 → height=5, width=2
// DRY RUN: [2,1,5,6,2,3] + sentinel [0]
// stack=[] (stores indices, increasing by height)
//   i=0(h=2): stack=[0]
//   i=1(h=1): 1

public int largestRectangleArea(int[] heights) {
    Deque<Integer> st = new ArrayDeque<>();
    st.push(-1); // sentinel for left boundary
    int max = 0;
    for (int i = 0; i <= heights.length; i++) {
        int h = (i == heights.length) ? 0 : heights[i]; // sentinel at end
        while (st.peek() != -1 && heights[st.peek()] >= h) {
            int height = heights[st.pop()];
            int width  = i - st.peek() - 1; // width between current and new top
            max = Math.max(max, height * width);
        }
        st.push(i);
    }
    return max;
}
